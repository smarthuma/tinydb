## 七、事务

> ⚠️ 注：本版本事务由 Python `db.transaction():` 块 API 触发，REPL 单条 SQL 走 autocommit 模式。

```bash
> CREATE TABLE users (id INT PRIMARY KEY, name TEXT NOT NULL, age INT);
> INSERT INTO users VALUES (1, 'alice', 30);
> INSERT INTO users VALUES (200, 'tx1', 1);
> SELECT * FROM users WHERE id = 200;
id  | name | age
----+------+----
200 | tx1  | 1
> .exit
```

✅ REPL 单语句路径：autocommit（每条 SQL 立即落盘）
✅ Python API 路径：`db.transaction():` 块支持 BEGIN/COMMIT/ROLLBACK（见 `scripts/functional_tests.py` 事务类别）

## 关键观察

1. **REPL 提示符**: `> ` (单 prompt) 一致
2. **空结果标识**: `(0 rows)` 准确区分空表 vs 无匹配
3. **NULL 显示**: Python 风格 `None`（不是 SQL 的 `NULL`）
4. **错误捕获**: `UniqueViolation` / `NotNullViolation` / `ParseError` 均以人类可读消息打印到 stderr，REPL 不退出
5. **CLI 一次性模式**: `-c` 支持多 SQL + 全部结果输出
