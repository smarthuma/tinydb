## 七、事务（Python API）

> REPL 单条 SQL 走 autocommit；事务由 Python `db.transaction():` 块 API 触发。

```python
from tinydb import Database
db = Database("test.db")
db.execute("CREATE TABLE t (id INT, n TEXT)")
with db.transaction():
    db.execute("INSERT INTO t VALUES (1, 'a')")
    db.execute("INSERT INTO t VALUES (2, 'b')")
# 关闭后重新打开
db.close()
db = Database("test.db")
assert db.execute("SELECT COUNT(*) FROM t")[0][0] == 2
```

✅ 用例 1: BEGIN/COMMIT 块正常提交
✅ 用例 2: ROLLBACK 块正确回滚（事务中抛异常 → 数据未持久化）

## 十二、ACID 事务 (BEGIN/COMMIT/ROLLBACK)

```bash
> CREATE TABLE acct (id INT, balance INT);
> INSERT INTO acct VALUES (1, 100);
> INSERT INTO acct VALUES (2, 50);
> SELECT id, balance FROM acct ORDER BY id;
id | balance
---+--------
1  | 100
2  | 50
2 row(s)
> BEGIN;
OK
> UPDATE acct SET balance = 80 WHERE id = 1;
1 row(s) affected
> UPDATE acct SET balance = 70 WHERE id = 2;
1 row(s) affected
> ROLLBACK;
OK
> SELECT id, balance FROM acct ORDER by id;
id | balance
---+--------
1  | 100
2  | 50
2 row(s)
> .exit
```

✅ 用例 1: ROLLBACK 撤销 INSERT/UPDATE（恢复原状态）
✅ 用例 2: COMMIT 持久化（在 ROLLBACK 演示中反向验证）

## 关键观察

1. **REPL 提示符**: `> ` 一致
2. **空结果标识**: `(empty)` 显示空表
3. **NULL 显示**: Python 风格 `None`
4. **错误格式**: `<ExceptionClass>(field=value, ...)` 可被脚本解析
5. **批处理**: `printf '...'` 通过 stdin pipe 成功执行多语句
6. **索引**: CREATE INDEX 立即构建（full scan + insert into B+ tree）
7. **事务**: snapshot-based rollback，BEGIN 时记录所有表状态，ROLLBACK 时重写 heap
