## 失败项详细分析

### ❌ #8 B-tree 索引

**proposal 说**："B-tree 索引" 在 Scope.In

**实测**：
```
> CREATE INDEX idx_t ON t (id);
expected keyword TABLE (line 1, column 8)
```

**根因**：
- 底层 `tinydb/index.py` 有完整 `BPlusTree` 实现（codec、seek/range/split/delete）
- 但 parser 不识别 `CREATE INDEX` 语法
- executor 也没有 `CREATE INDEX` 处理路径
- 索引仅在 test_index.py 中作为独立 API 被测

**修复方向（v0.2 计划）**：
1. parser 加 `CREATE [UNIQUE] INDEX` 分支
2. executor `_exec_create_index` → `BPlusTree.create(store, key_type)`
3. 写入 catalog.indexes 映射
4. 集成测试：EXPLAIN 显示索引路径

### ⚠️ #10 ACID 事务

**proposal 说**："ACID 事务：BEGIN、COMMIT、ROLLBACK" 在 Scope.In

**实测 REPL 行为**：
```
> BEGIN;          ← 返回 OK，但没真正开 tx
> INSERT ...;     ← autocommit，立即落盘
> COMMIT;         ← 返回 OK，但没真正提交（autocommit 没"提交"概念）
```

**根因**：
- `tinydb/tx.py` 有完整 `TxManager` 实现（BEGIN/COMMIT/ROLLBACK + WAL append + fsync）
- CLI 简单路由 `BEGIN/COMMIT/ROLLBACK` 到 `TxManager`，但**没有把 executor 的写操作包在 transaction 内**
- executor 自身仍走 autocommit 路径

**修复方向（v0.2 计划）**：
1. `Executor.__init__` 接受 `tx_manager` 参数
2. `Executor.execute()` 在 `_current_tx_id` 非空时记录 mutation 到 WAL
3. CLI 走 `tx_manager.begin()` → `executor.execute()` → `tx_manager.commit()` 三步
4. 集成测试：`BEGIN; INSERT; ROLLBACK;` 后 SELECT 看不到数据

## 关键观察

1. **proposal 描述 12 项**，**10/12 完整实现** — 这是基本盘的胜利
2. **2 项 gap 都是 v0.2 计划**（与 `docs/CHANGELOG.md` 中的 Known Limitations 完全一致）
3. **底层代码已存在**（BPlusTree、TxManager），只是**未在 SQL 层面/REPL 层面接线**
4. **没有实现"超出 scope"** 的功能（无 JOIN、无并发、无网络） — Scope.Out 严格遵守

## 结论

**proposal 中的 12 项 Scope.In 功能，10 项已完整交付，2 项存在可修复 gap。**
